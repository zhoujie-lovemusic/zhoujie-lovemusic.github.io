10.watchEffect

在上面的例子中，我们在 watchEffect 的回调函数中依赖了 state.count，因此当 state.count 发生变化时，会触发 watchEffect 的回调函数。

然而，在 increment 函数中虽然修改了 state.count 的值，但由于 watchEffect 并没有依赖到 increment 函数，所以不会触发 watchEffect 的回调函数。

因此，只有在 watchEffect 函数内部依赖的响应式数据发生变化时，才会触发 watchEffect 的回调函数。其他的响应式数据变化不会对 watchEffect 产生影响。

12.自定义hooks

  • 定义：是对组合式API的封装，写在setup中代码更加清晰
  • 类比：相当于Vue2中的mixin

13.toRef/toRefs

  • 使用场景：想单独对某个对象的某些属性进行响应式更新，但是不想写多个对象.属性

14.shallowReactive和shallowRef

shallowReactive：只针对对象数据第一层进行响应式

shallowRef：可以对基本数据类型进行相应，但是对于对象数据不响应（没有求助reactive用proxy实现）

使用场景：针对对象数据类型，可以提高性能

  • 如果某个对象层次很深，但是明确知道只有第一层会用到响应式——–shallowReactive
  • 如果某个对象只可能在将来被整体替换，并且它里面的属性不会被修改——–shallowRef

​ （shallowReactive只是只针对第一层，而shallowRef都没有进到对象里面，只考虑重新赋值后的属性初始值）

15.readOnly和shallowReadonly

把数据变成只读的（ref、reactive），shallowReadonly可以针对对象类型只在第一层限制只读

（readOnly对于普通对象还可以转化为proxy对象，谁叫这是vue3的api）

使用场景：某数据在组件A，传递给组件B时，要求组件B只能使用不能修改

16.toRaw和markRaw

toRaw：可以将reactive对象———>原对象

场景：临时不需要响应式——— 用的非常少

markRaw：永久标记成非响应式的

场景：第三方库不需要响应式 或者 某个某个对象层数太多，非响应式可以提高性能

24.其他

1.全局API的转移

  • Vue 2.x 有许多全局 API 和配置。
    • 例如：注册全局组件、注册全局指令等。

Vue3.0中对这些API做出了调整：

  • 将全局的API，即：Vue.xxx调整到应用实例（app）上

    2.x 全局 API（Vue） 3.x 实例 API (app)
    Vue.config.xxxx app.config.xxxx
    Vue.config.productionTip 移除
    Vue.component app.component
    Vue.directive app.directive
    Vue.mixin app.mixin
    Vue.use app.use
    Vue.prototype app.config.globalProperties

2.其他改变

① data选项应始终被声明为一个函数（避免对象形式的引用关系互相影响）
② 过渡类名的更改

Vue2.x写法

Vue3.x写法

③ 移除keyCode作为 v-on 的修饰符，同时也不再支持config.keyCodes（因为兼容性差）
④ 移除v-on.native修饰符
  • 父组件中绑定事件

子组件中声明自定义事件

⑤ 移除过滤器（filter）

过滤器虽然这看起来很方便，但它需要一个自定义语法，打破大括号内表达式是 “只是 JavaScript” 的假设，这不仅有学习成本，而且有实现成本！建议用方法调用或计算属性去替换过滤器。
