前端开发经验(持续更新)

查漏补缺：

技术部分

URL和路由的区别

URL是整个地址，而路由只是端口号后面的部分

日期的比较

需要先将字符串转化为日期对象才可以直接用 ><比较

path参数和query参数的区别

两者的区别：前者没有传递参数，仅仅是一个URL，后者会传递query参数，比如?couponId=5

post请求也可以有查询参数

get请求 = post请求在URL后面直接拼接参数（post(/sourceadmin/manager/new/updateVipPurviewByVipId.action?vipId=${data.vipId},data)），因为get请求默认就是query参数

打包相关

NODE_OPTIONS 报不是内部命令时，执行 npm install -g win-node-env 再尝试

git

git如果改的东西不多还是建议，先pull再commit，还是想清晰地显示下 commit的内容

开发经验

后端 数据的 问题，前端 提醒一下就行了，我该干嘛干嘛，不用等他，我前端逻辑又没错

h5和vue目录的关系

纯H5项目的文件就是Vue项目build之后的文件，只保存用到的css、js、html

前后端不同环境的区别

前端不同环境（看网址URL）决定了页面不同，而后端不同环境（看接口URL）决定了数据的不同

Bug修改

若接口有数据且也能正确获取到，但就是 不显示 ——-> 看看控制台有没有错误，只要有某行代码有问题 阻塞下面 代码的执行

vue项目目录结构

–src

assets：静态文件——–图片、图标

api：api.js，注意要按模块分：common.js、xxx.js…；config.js，为api.js配置api环境、请求拦截、封装请求方法…

components：公共组件，即封装的 可复用的组件

plugins：第三方插件，按需Vue.use

layout：布局组件，主页面的 轮廓组件：SideBar、Navbar

router：路由文件，注意也要 按模块分，子菜单注意要用children，避免单个路由写的过长

store：仓库，注意要 按模块分

utils：自己封装的js工具，登陆加密、数据格式转换

–root

main：入口文件，注册插件、组件、原型上添加方法等

.env.debug/development/production：设置Api调用环境的地址

vue.config.js：打包的配置、优化等

封装

项目比如后台管理系统，对于表单搜索和表格展示较多，需要对 表单和表格进行封装
