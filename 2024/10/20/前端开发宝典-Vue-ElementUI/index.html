前端开发宝典(Vue+ElementUI)

Js

看到UI稿就要立马在脑里设计好数据结构

  • 相似块不断出现 ———- > 数组
  • 每个块又有各自结构 ———-> 对象

tableData的数据结构要和html展示结构要一致

  • 如果数据层数太多的话，会出现无法编辑的情况

    比如当前el-table-column的字段是currentTicketTypes，那么就不能跳过currentTicetTypes直接用下一层数据realName，结果就是el-input无法输入，卡住了一样

：符号的意思

  • 不仅仅是能填变量，而是指可以填常量，看你原来是什么数据，就是什么。比如:label=”3”，此时获取到的值就是3，而label:”3”，获取到的是“3”

多用const而不是let

  • 中间处理的数据都 重新用const定义一个变量，然后再赋值给想要的数据上，这样各司其职很清晰

Async/Await

异步数据作为函数的返回值来使用

  • 想要获取类似接口这样的异步数据作为函数的返回值在别处使用，必须使用await/async，因为只有await才能把数据返回出去
  • 而async函数返回的一定是一个Promise，因此在别处使用又要用await
  • async一定还要加在离当前await最近的函数上，如果是回调函数要加在回调函数前面

异步操作的遍历不能用map函数

  • 异步操作的遍历不能用map函数，否则返回的每一个item是一个promise，即使使用了await关键字也无法等待，因为map不回去处理异步操作，比较急躁

  • 要用for循环（item的某一个属性值单独用await，然后再赋值上去）或者用Promise.all所有的promise来解决（用了map之后，是一个包含promsie的数组，那么就用Promise.all获取所有结果）

  • ```js
    for (const item of this.reserveTemplateInfo) {

            const reservePeopleAll = (await this.getReservePersonTypeOptions(item.id)) || [];
        reserveObjectPerson.push({
          name: item.name,
          id: item.id,
          reservePeopleAll,
          reservePeople: [],
          department_type: 0,
        });
      }
      this.formData.reserveObjectPerson = reserveObjectPerson;

  • every：返回值为boolean值，用于判断，数组的每个item都满足条件才返回true（有点类似allSettled）

  • some：返回值为boolean值，有一个满足返回true

  • concat：多个数组连成一个数组，在reduce方法里比较常用

对象

  • hasOwnProperty：可以用来判断对象是否有传入的属性值

CSS

内容过多时出现滚动条

  • 在父级元素（哪个父级元素看需要滚动条多大范围）上增加 overflow:auto + 固定width

元素定位不相互挤压

  • 只要用margin就会在文档流里，就会相互挤压，而绝对定位还能脱离文档流

Vue

数据中间处理不要在响应式数据中进行

  • 避免不必要的响应到页面上的情况
  • 重新用const定义一个变量来给这个响应式数据赋值

动态修改Html结构-v-html

  • 可以在原本要显示数据的地方使用v-html指令，数据用个计算属性（提供一个计算的平台）

  • 本质上是用html字符串的处理，那么就可以用各种字符串处理方法，不好处理用split转成数组处理，最后在拼接（join方法）

  • 找到文字内关键词并标绿

模态框（dialog）作为子组件显示，可见性需要更新回父组件

  • 子组件要使用计算属性接受，顺便利用set来更新回去

  • 不是单向数据流吗？为什么要更新回去？因为父子组件各自都有权力控制显示，不更新回去任一方的更新都将无效，且这样不用

  • ```js
    computed: {

    localExportFailVisible: {
  get() {
    return this.exportFailVisible;
  },
  set(val) {
    this.$emit('update:exportFailVisible', val);
  },
},
