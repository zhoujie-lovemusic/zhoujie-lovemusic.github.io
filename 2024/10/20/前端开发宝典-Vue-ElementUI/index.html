前端开发宝典(Vue+ElementUI)

Js

tableData的数据结构要和html展示结构要一致

  • 如果数据层数太多的话，会出现无法编辑的情况

    比如当前el-table-column的字段是currentTicketTypes，那么就不能跳过currentTicketTypes直接用下一层数据realName，结果就是el-input无法输入，卡住了一样

CSS

内容过多时出现滚动条

  • 在父级元素（哪个父级元素看需要滚动条多大范围）上增加 overflow:auto + 固定width

Vue

数据中间处理不要在响应式数据中进行

  • 避免不必要的响应到页面上的情况
  • 重新用const定义一个变量来给这个响应式数据赋值

ElementUI

el-table

设置el-table每个单元格高度

  • 去掉padding影响

  • 通过行高设置具体高度（去掉padding之后此时内部是span高度由行高决定）

x轴出现滚动条

  • 可以在每个item加固定width

el-dialog

自定义内间距，且内容被子元素高度撑开

  • 直接设置.el-dialog__body 的padding（使用deep加速器）
  • 如果内容不想被撑开，想加上滚动条，在子元素外部添加overflow:auto + 固定width

e-form

想要校验的“红星”样式，但是不触发任何校验

  • 直接在el-form-item加上class=”is-required”

radio-group

循环radio-group时的注意点

  • 要保证每个radio-group有自己的v-model值，这样各个radio-group的选中的数据不会相互影响

业务场景

无感刷新

  • 用setInterval每隔一秒调用获取列表的接口（记得组件销毁时clearInterval）———–刷新
  • “删除”、“编辑”等修改列表数据的操作时需要clearInterval，操作成功后再开启 —- 保持无感刷新
  • 把列表加载数据时的loading效果要去掉 ——— 无感

template-admin框架的下拉框独有的回显方式

  • 内置有useProxy方法获取某个下拉框数据的（字典值和label）组成的数组
  • 这个字典值会默认从store中去取，没有还会自动去请求接口并保存在store中
  • 如果是 修改 模式，则在具体的字典值来之前准备好数组即可；如果是 查看 模式，则还需要另外的方法将字典值转化为对应的label值

下载切图时一定要用svg格式的，避免分辨率不同带来的模糊

  • svg格式的图片能自适应还不失真
