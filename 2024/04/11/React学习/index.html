React学习

前言

由于本人之前已有扎实的Vue框架基础，本篇将会通过与Vue框架的对比来达到快速学习的目的

npx

npx和npm相比，除了和npm一样都可以下载Packages之外，还有以下区别：

安装范围小：用npx安装的包是临时安装的，用完之后会自己删除

功能多样：还可以 直接执行 下载的package中的可以执行脚本，例如：npx mocha

最常用的操作：npx create-reeact-app my-app（创建的同时执行项目）

充当nvm的功能，指定nodejs的版本，例如：npx node@12.20.0 -v

总结：开发者希望npx一个工具方便多个用途

  • 安装依赖
  • 不让包污染磁盘空间，保证 包在磁盘中只有一份且包是最新的
  • 如果不要最新的包，还可以用它来改版本

什么是JSX？如何解析的？

JSX是Js和XML（HTML）的缩写，指它能 将js和html包裹在一起（html在js内部），这样既有了HTML的声明式模板的能力，也有了js的可编程能力

它 是 通过解析工具Babel 使jsx文件转化为 浏览器可以识别的文件

React是如何处理js表达式的？

我们知道在Vue中是用插值语法是双括号，而React中是一个括号，里面可以放js表达式，比如常量、变量、有返回值的函数，还有对象，写对象和Vue不一样

如何实现列表渲染？

使用js中map方法作为一个js表达式嵌入到{}中

如何实现条件渲染？

简单逻辑

利用 && 或 三元运算符

复杂逻辑

利用 if语句/ switch语句写在函数里返回，然后调用

事件绑定

要点如下：

  • 事件命名规则：on+事件名 ———- 区分于 Vue
  • 函数也是js表达式，写在 {} 内 ———- 区分于 Vue
  • 传参是正常的js规则
    • 一定要写成箭头函数：为了和函数定义保持一致，书名只是个结构，（）用于传参，千万不要直接写函数名，那是直接调用（都不会区分什么时候触发事件）
    • 要获取当前事件对象 ———- 直接写函数名即可（不要写括号，那是调用），因为默认有参数$event
    • 既要事件对象 又要 自定义参数，则（这里是存事件对象的）=>fun(e,自定义参数)

组件

  • 和函数声明一样
  • 首字母要大写，这个是 和 函数做区分的 依据

useState

基础使用

  • 用来设置响应式数据，有点像Vue3的ref、reactive，左边是变量名，右边是值
  • xxx,setXXX是响应式数据配套的，后者用于修改

修改状态的规则

  • 得用setXXX包装的方法来修改才能触发响应式更新，手动更新是没用的 — 内部方法实现了响应式
  • 用新数据替换而不是在原来的基础上修改 —– 就是为了避免不用setXXX来修改数据

复杂对象State的使用

  • 用全新的对象替换->用…和新的属性，结合来使用

样式写法

  • 行内
    • 可以单独把样式对象写在外面
  • 类名
    • 写在css文件里在js文件里引入
    • 类名尽量 原子化（类名 见名知意、一个类选择器只有一个样式、方便复用）

demo练习-导航Tab点击出现高亮

优化版-类名使用classnames插件

参数一：静态类名，参数二：{动态类名：条件}

表单输入框双向绑定

  • 就是v-model的原理，数据驱动+事件绑定

Ref的使用

  • React中没有this对象，只能从外部导入钩子函数，利用它的current属性获取dom对象

组件通信

父子通信（props）

内容

  • 由 :xxx -> xxx={}
  • 和Vue3的setup一样从形参中去拿props属性
  • 默认不让子组件修改父组件数据

结构

  • 类似Vue2中的插槽,，只是React把它作为children放进了props对象内

子传父

  • 对比Vue，没有了 :on、:emit这种关键字

  • 传递的回调也是放在props中的

兄弟组件传值

状态提升：在共同的 父组件 中设置响应式状态，接收A的数据->传递给B

  • 状态：A传递的数据修改，B也能相应
  • 提升：提升到父组件中

跨层机制（Context）

  • 用createContext创造一个对象xxx，首字母要大写
  • 顶层组件使用xxx.Provide 将底层组件包裹
  • 在底层组件中使用useContext获取

useEffect

基本使用

简单来说，就是Vue中 mounted+updated+watch 的结合，Vue3中的watchEffect借鉴了useEffect。

参数一：回调函数，参数二：依赖项

  • 当没有传第二个参数 —- > mounted+updated
  • 传入第二个参数 —-> mounted+watch
  • 第二个参数决定了 渲染完成后续 执行回调的时机

副作用清除

  • 在 ueseEffect 内部 再返回一个新的函数，在这里面写清楚副作用逻辑
  • 原因：ueseEffect 内部返回的函数 通常是在组件卸载时 执行，因此可以在这里写清楚副作用逻辑

自定义hook

  • 就是 自定义逻辑 抽取和封装，熟悉一下 hook 的使用方式即可

Reacthook的使用规则

  • 只能写在组件内或自定义hooks内
    • 自定义hooks要写成功能难免会调用现成的hook
    • 后续都是 组件化 开发，基本不会写在组件外
  • 只能写在组件或自定义hooks 的顶层，不能写在if、for等逻辑的内部
    • hooks说白了就是一个功能的 封装，想想js中功能模块化，在使用时也是在最上面 Import 的

Redux

流程

使用流程不复杂，但是写法复杂，下面流程总结下：

1.准备好子模块的store

2.将子模块引入总Store

3.总store安装到App上

4.使用useDispath发送具体请求

Redux的命名由来？

re->react

dux->受到Flux（一种架构）的启发，它就是这样管理数据的

reducer的命名由来？

er->是Redux的一个子项，因此引入总Store的关键成分就是reducer

必备的@reduxjs/toolkit和react-redux包的作用是什么？

@reduxjs/toolkit：redux工具包，里面有很多工具函数，可以让redux使用起来方便、有条理

react-redux：用于连接 react项目和redux，因为redux原本是可以单独使用的
